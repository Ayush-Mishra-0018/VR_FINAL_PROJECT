1. Confirmation Gate (Most Important Rubric Gap)
The rubric explicitly requires:
"Confirm crop / Re-crop before proceeding".

Current behavior:
- selecting a region immediately enables retrieval.

Required fix:
- after selecting a YOLO region, show:
    [ Confirm Crop ]
    [ Re-select Region ]
- ONLY after confirmation should:
    - CLIP encoding run
    - FAISS retrieval run

Goal:
Clearly demonstrate:
YOLO detection → user verification → retrieval pipeline.

Do NOT redesign UI.
Keep current region cards and selection flow intact.


2. Re-Crop / Re-Selection Support
The rubric explicitly mentions:
"Re-crop".

Current behavior:
- user can only select among YOLO detections.

Required fix:
Add a lightweight recovery flow:
- "Re-select Region" button
OR
- "Back to Detection" flow

The user must be able to:
- reject current crop
- choose another detected region
before retrieval starts.

Do NOT implement a heavy manual crop editor unless simple.
A clean re-selection flow is enough.
